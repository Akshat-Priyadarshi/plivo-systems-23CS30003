1. Architecture Overview

The system is built in C++ and uses Forward Error Correction (FEC) over UDP to survive a hostile network while staying under the 2.0x bandwidth budget.

2. Sender Logic

The sender reads strictly 164-byte packets from the harness. Instead of a naive 1:1 forward, it implements a 2:1 XOR Parity scheme. It transmits two standard data packets (Type 0) and computes an XOR combination of their payloads to send as a third parity packet (Type 1). This yields a highly optimized 1.55x bandwidth overhead.

3. Receiver Logic

The receiver utilizes a non-blocking poll() loop to read incoming packets into a std::map jitter buffer. If a data packet is dropped by the relay, the receiver uses the subsequent XOR parity packet to mathematically reconstruct the missing payload on the fly. A rigorous playout loop ensures packets are flushed to the harness player strictly before their t0 + DELAY_MS + i*20ms deadline expires.
